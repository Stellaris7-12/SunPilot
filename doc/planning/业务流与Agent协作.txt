发单Agent / 回单侧多Agent = 业务大脑
PageAgent = 页面执行员 + 可视化操作员
Mock Tools / MySQL = 业务系统与证据底座
人工坐席/处理员 = 最终责任人

通话记录
  -> 发单Agent：提取业务字段、生成工单草稿
  -> PageAgent：打开/识别发单页面，移动鼠标，自动回填字段，一键提交
  -> 多Agent：分类、抽取、工具调用、风险判断、生成回单
  -> PageAgent：打开工单处理页，定位证据，回填处理意见和客户回单，执行高频结案动作
  -> 人工/系统门禁：高风险确认、异常接管、最终审计


“客服电话结束后发单Agent发单 -> PageAgent 辅助填单 ----解耦----- 回单Agent（多 Agent ）处理 -> PageAgent 辅助填单/回单 -> 结案复核”  让Pageagent贯穿全流程


PageAgent的功能：承接前面的多Agent处理后的数据，识别当前页面的元素，找到填单的位置，操作相应按钮、选项、以及文本框填写
