# Code-X·极协——信用卡一站式服务引擎 需求分析文档


## 一、项目概述

### 1.1 项目名称
Code-X·极协——信用卡一站式服务引擎

### 1.2 项目定位
依托“CodeX编码智能体+大模型Agent”双引擎技术底座，以打通全域业务系统与全量会话数据为基础，打造信用卡行业工单智能运营的标杆模式，实现客户工单问题全流程自动化处置与运营持续自优化。


## 二、项目背景与现状分析

### 2.1 业务规模与现状

| 维度 | 现状数据 |
|------|----------|
| 年度工单总量 | 330万+件 |
| 业务场景覆盖 | 700+类 |
| 涉及部门 | 300+个部室 |
| 涉及人员 | 近万名员工 |
| 单张工单平均处理时长 | 约30小时 |
| 客户当日未解决转化率 | T日活跃客户中35%进入线上处理，其中约8%需发工单跟进 |
| 工单系统发单占比 | 65%通过原生页面发起 |
| 工单数据来源 | 100%需业务数据+对话数据生成（71.62%来自业务系统接口，28.38%需人工补充对话内容） |

### 2.2 核心痛点

**（一）客户侧痛点**

1. **问题未一次解决**：客户经历层层深入交互后问题仍未解决，体验持续受损。
2. **等待周期过长**：单张工单从发单到回复全流程约30小时（含等待、审批时间），与客户预期严重不符。

**（二）内部运营痛点**

1. **体量与复杂度高**：工单规模超330万件/年，覆盖700余类业务，涉及300余个部室、近万名员工，管理难度极大。
2. **重复劳动占比高**：人工需跨系统调取、复制填写业务数据与对话数据，操作冗余严重。
3. **入口与自助承接不足**：大量本可自助处理的诉求仍通过人工进线发单，挤占坐席服务资源。
4. **发单环节效率受限**：700余类长尾工单尚未完成业务系统嵌入，坐席需从业务系统摘抄数据、从对话中梳理诉求，发单效率低。
5. **回单环节标准不一**：回复依赖处理人员个人经验与知识检索能力，质量与时长差异大；高频重复工单仍需逐单人工判断与回复，人力浪费严重。


## 三、项目目标

### 3.1 总体目标

以“对外提升客户体验、对内减少运营损耗”为核心，以“工单全链路重构+人工操作极简化”为终极方向，实现三大核心任务：

| 任务维度 | 目标方向 |
|----------|----------|
| 进线前 | 在各自助渠道完善解决方案，实现自动化分流 |
| 进线中 | 打通Agent能力，提升客户问题一次解决率 |
| 进线后 | 大幅减少发单、回单环节人工操作，最大程度消除人工干预环节 |

### 3.2 量化目标

| 阶段 | 关键指标 | 目标值 |
|------|----------|--------|
| 第一阶段（试点） | 发单效率提升 | 60% |
| 第二阶段（全量覆盖） | 一键发单覆盖率 | 80%以上 |
| 第二阶段（全量覆盖） | 工单全流程处理时长缩短 | 40% |
| 第二阶段（全量覆盖） | 运营配置效率提升 | 70% |
| 全量目标 | 单张工单人工处理时长平均压缩 | 50% |
| 头部场景试点 | 节省人力 | 70人/年 |


## 四、业务需求

### 4.1 业务范围

**（一）覆盖系统**
项目涉及13个业务系统，覆盖近100个业务节点，涵盖但不限于：

- 业务数据系统（辅助定位、行销、客控等132张工单下的44个字段）
  - 收件编号、客户信息、申请资料、处理状态
  - A/B级征信代码、进度节点、处理结果
  - 卡片信息、进度明细、名单状态、最后联系状态
  - 收件状态、推广人员、推广单位等
- 对话数据系统
  - 申请进度查询、加急申请、补充资料、取消申请等场景
- 业务系统内嵌工单发起入口

**（二）覆盖业务场景**
- 全量700余类工单场景，试点阶段优先覆盖：
  - 协商还款
  - 伪冒管制
  - 申请进度查询
  - （上述三类占总单量约70%）

### 4.2 业务流程需求

**（一）进线前——溯源分流**

在客户进线前，通过各自助渠道（掌上生活APP、IVR、官方微信等）完善解决方案，实现自动化分流，降低人工进线量。

**（二）进线中——一次解决**

坐席服务过程中，通过Agent能力辅助实时解决客户问题，减少需发单跟进的比例。

**（三）进线后——智能发单与回单**

| 环节 | 核心需求 |
|------|----------|
| 发单侧 | C端自助发单（掌上生活、IVR、微信等全自助渠道）；B端业务数据自动回填、对话摘要自动生成、业务系统一键发单 |
| 回单侧 | 减少多系统来回切换；智能回单推荐与知识自动匹配；高频工单自动化判断与回复 |

**（四）全链路管理**

构建全局工单旅途可视化管理体系，实现从“结果式管控”向“全流程节点式精细化治理”升级。

### 4.3 非功能需求

1. **数据安全**：涉及客户征信信息、账户信息等敏感数据，需严格权限管控与数据加密。
2. **系统集成**：需与13个存量业务系统、全渠道会话数据平台无缝对接。
3. **高可用性**：作为核心运营支撑系统，需保障高可用性与稳定性。
4. **可扩展性**：架构需支撑横向扩展至30+类跨条线协同场景。
5. **数据准确率**：试点阶段需验证方案可行性与数据准确率达标。


## 五、用户需求

### 5.1 用户角色与核心诉求

| 用户角色 | 核心诉求 |
|----------|----------|
| **客户** | 问题快速响应、一次性解决、无需重复沟通、减少等待时间 |
| **坐席人员** | 发单操作极简化（免摘抄、免梳理、一键提交）、减少跨系统切换 |
| **工单处理人员（后台）** | 回复效率提升（自动回填、智能推荐）、减少重复劳动、提升回复质量 |
| **运营管理人员** | 工单全局可视、节点级管控、异常预警、数据驱动决策 |
| **技术运维人员** | 系统稳定运行、配置灵活、故障快速定位与恢复 |

### 5.2 用户场景示例

1. **客户场景**：客户通过掌上生活APP自助发起申请进度查询工单，系统自动调取业务数据、生成工单并流转，无需坐席介入，客户实时获取处理进度。
2. **坐席场景**：通话中坐席通过业务系统内嵌发单按钮，系统自动回填业务数据、自动生成对话摘要，一键提交工单，全程无需人工摘抄填写。
3. **处理人员场景**：处理人员打开工单，系统自动推荐处理方案与知识匹配项，高频场景自动完成回复与结单。


## 六、技术需求

### 6.1 技术架构

项目采用“双引擎驱动”技术架构：

| 组件 | 说明 |
|------|------|
| **CodeX编码智能体** | 负责业务系统接口调用、数据映射与自动填充、跨系统操作执行 |
| **大模型Agent** | 负责对话理解、意图识别、摘要生成、知识匹配、回复推荐 |
| **AI运营管理闭环** | 贯穿全流程的监测分析与自优化机制 |

### 6.2 功能模块需求

**（一）Agent核心功能**

- **发单Agent**：
  - 自动识别业务场景与工单类型
  - 从13个业务系统自动调取所需字段数据
  - 自动生成客户对话摘要
  - 一键完成工单创建与分发

- **回单Agent**：
  - 自动匹配处理知识与历史案例
  - 生成推荐回复内容
  - 高频重复工单自动判断与处理
  - 跨系统数据自动查询与回填

- **分发/跟踪Agent**：
  - 智能路由至最合适的处理团队
  - 全流程节点状态追踪
  - 异常情况自动预警

**（二）AI运营监测分析平台**

| 功能模块 | 需求说明 |
|----------|----------|
| 核心指标监测 | 工单量、处理时长、一次解决率、人工参与度等实时看板 |
| 异常预警 | 流量突增、处理超时、系统异常等自动告警 |
| 根因分析 | 工单积压/效率下降的自动归因分析 |
| 自动化配置 | 工单SOP流程配置、Agent策略动态调整 |

### 6.3 关键接口需求

- 与13个存量业务系统接口对接，实现数据实时读取与写入
- 与全渠道会话系统对接（语音、在线、APP等）
- 与工单系统原生能力对接
- 与知识库系统对接

### 6.4 数据需求

| 数据类别 | 需求说明 |
|----------|----------|
| 业务数据 | 132张工单下44个字段的标准化映射与自动填充能力 |
| 对话数据 | 全量会话数据的实时接入与语义理解处理 |
| 工单SOP数据 | 700余类工单的标准作业程序梳理与数字化建模 |
| 运营指标数据 | 全链路节点数据的采集、存储与分析 |


## 七、项目阶段规划

### 7.1 第一阶段：试点验证

**覆盖范围**：协商还款、伪冒管制、申请进度查询3类高频工单（占总单量约70%）

**主要工作内容**：

- 3类工单SOP梳理、Prompt调优、业务系统接口对接
- 业务系统内嵌发单功能上线
- 坐席端陪伴式发单Agent上线
- 智能回单辅助基础功能上线
- 核心指标运营监测模块搭建

**目标**：试点工单发单效率提升60%，验证方案可行性与数据准确率

### 7.2 第二阶段：全量覆盖

**覆盖范围**：全量700余类工单

**主要工作内容**：

- 全量工单SOP梳理、数据映射与功能上线
- 业务系统内嵌发单全覆盖
- 掌上生活、IVR自助发单入口上线
- AI运营监测分析平台完善（异常预警、根因分析、自动化配置）
- 全量坐席与处理人员培训推广

**目标**：80%以上工单一键发单，全流程处理时长缩短40%，运营配置效率提升70%

### 7.3 第三阶段：生态扩展

**覆盖范围**：客经、市场、催收、行销等30+类跨条线协同场景

**主要工作内容**：

- 技术架构与运营模式横向复制
- 全矩阵Agent协同智能运营体系构建

**目标**：价值辐射全卡中心所有跨部门协同场景


## 八、项目价值预期

### 8.1 体验提升价值

提升客户问题“一次解决率”，缩短工单处理等待周期，优化客户服务体验。

### 8.2 效率提升价值

| 维度 | 预期成效 |
|------|----------|
| 发单侧 | 自助渠道降低人工成本，业务数据自动回填+对话摘要自动生成+一键发单，消除重复操作 |
| 回单侧 | 减少跨系统查询切换，智能推荐提升回复准确率，单张工单处理时长平均压缩50% |
| 管理侧 | 节点式精细化治理替代粗放式结果管理 |

### 8.3 复用拓展价值

项目技术架构与运营模式具备强可复制性，可快速延伸至30+类跨条线协同场景，打造全矩阵Agent协同智能运营体系。

### 8.4 绿色运营价值

减少重复人工操作与跨系统冗余流转，降低办公能耗，践行低碳运营理念。
