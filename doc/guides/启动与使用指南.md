# TicketAgent 启动与使用指南

> 版本：模块 M PageAgent 演示版  
> 更新日期：2026-07-22

## 一、环境要求

| 依赖 | 要求 | 说明 |
|---|---|---|
| Python | 3.10+，低于 3.13 | 后端 FastAPI 与 Agent 链路 |
| uv | 建议使用当前项目锁文件 | Python 依赖管理 |
| Node.js | 18+ | 前端 Vue/Vite |
| npm | 9+ | 前端依赖与构建 |
| MySQL/TDSQL | MySQL 兼容协议 | 主演示数据库 |

当前数据库口径：

- 主演示环境使用 MySQL/TDSQL，不再支持 SQLite 运行路径。
- 开发/演示库为 `ticket_agent`，自动化 smoke 测试库为 `ticket_agent_test`。
- 前端不直接连接数据库；前端访问 `http://localhost:8000/api`，后端再通过 Repository 访问 MySQL/TDSQL。
- 工单写操作、状态流转、操作日志、Mock 业务域数据、工具审计和通话发单接口均通过后端 Repository/API 访问。

## 三、数据库配置

### MySQL/TDSQL 模式（推荐演示）

不要把真实密码写入仓库。当前约定 MySQL 用户为 `root`，密码来自系统环境变量 `MYSQL_ROOT_PASSWORD`。

PowerShell 临时配置：

```powershell
$env:MYSQL_ROOT_PASSWORD="your-password"
$env:DB_BACKEND="mysql"
$env:DATABASE_URL="mysql+asyncmy://root:${env:MYSQL_ROOT_PASSWORD}@127.0.0.1:3306/ticket_agent?charset=utf8mb4"
```

如果系统环境变量里已经有 `MYSQL_ROOT_PASSWORD`，只需要设置：

```powershell
$env:DB_BACKEND="mysql"
$env:DATABASE_URL="mysql+asyncmy://root:${env:MYSQL_ROOT_PASSWORD}@127.0.0.1:3306/ticket_agent?charset=utf8mb4"
```

MySQL/TDSQL schema 文件：

```text
ai-engine/migrations/mysql/001_i1_schema.sql
```

后端启动和 smoke 会自动创建缺失的 MySQL 数据库；执行前确认目标库是本地演示库或可重建沙盘库，不要指向真实业务库。

## 九、使用流程

1. 打开 `http://localhost:5173/tickets`。
2. 在“通话发单工作区”选择通话样本，点击“生成发单草稿”，确认字段来源和缺失提示。
3. 点击“PageAgent 可见发单”，观察 PageAgent 高亮页面、填写草稿并提交；成功后跳转到 `/tickets/:id`。
4. 在详情页右侧 PageAgent 控制台点击“自动回单”，它会触发多 Agent 处理、定位证据、填写并保存客户回单。
5. 中高风险、缺字段或异常场景会停在人工补充/确认/复核节点；结案仍必须走 `/api/tickets/{id}/close`。

注意：PageAgent 是受控 GUI 执行层，只能使用白名单页面动作；业务判断仍由后端五类 Agent 和 Mock Tools/证据链完成。

## 十一、常见问题

**Q: 前端页面为空或请求失败？**

A: 确认后端运行在 `localhost:8000`。前端 API 客户端默认访问 `http://localhost:8000/api`。

**Q: PageAgent 按钮点了没有反应？**

A: 先刷新 `http://localhost:5173/tickets`，确认前端已加载最新布局。正常情况下“生成发单草稿”和“PageAgent 可见发单”按钮不应被草稿表单覆盖。

**Q: 为什么启动时报 MySQL 配置错误？**

A: 模块 K 后默认就是 MySQL。请确认系统环境变量 `MYSQL_ROOT_PASSWORD` 存在，或显式设置 `DATABASE_URL` 指向本地 `ticket_agent`。

**Q: MySQL 中没有 30 条干净演示工单？**

A: 确认连接的是 `ticket_agent` 演示库，然后执行：

```powershell
$env:DB_BACKEND="mysql"
$env:DATABASE_URL="mysql+asyncmy://root:${env:MYSQL_ROOT_PASSWORD}@127.0.0.1:3306/ticket_agent?charset=utf8mb4"
.venv\Scripts\python.exe ai-engine\scripts\reset_demo_data.py --confirm-reset-demo
```

**Q: I1/I2/I3 是否会污染演示库？**

A: 不会。模块 K smoke 会自动使用 `ticket_agent_test`，演示库 `ticket_agent` 只通过受控重置脚本或后端启动 seed 更新。

**Q: 能否直接用工单编号打开详情？**

A: 可以。`/tickets/:id` 同时支持内部 id 和可见工单编号，例如 `/tickets/T202607210001`。

**Q: `closedLoopSuccessRate` 是真实生产结案率吗？**

A: 不是。它当前表示“状态/预期结果匹配率”，用于工程回归评测，不代表真实客户结案率。

**Q: LLM 返回 401 或鉴权失败？**

A: 设置有效的 `DEEPSEEK_API_KEY`，或确认 `LLM_BASE_URL`、`LLM_MODEL` 与当前供应商兼容。
