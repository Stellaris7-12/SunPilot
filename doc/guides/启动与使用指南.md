# TicketAgent 启动与使用指南

更新日期：2026-07-22

本指南覆盖整个 TicketAgent 项目：

```text
通话记录 -> 发单 Agent -> 标准工单 -> 多 Agent 处理 -> Mock Tools 审计 -> SunPilot/PageAgent 辅助填单回单 -> 人工复核结案
```

依赖要求：Python 3.10+ 且低于 3.13、uv、Node.js 18+、npm 9+、MySQL/TDSQL。

## 2. 配置方式

不要把真实 API Key 写入仓库文件。项目统一通过 [ai-engine/config.py](C:/Users/heyunhui/PyProjects/TicketAgent/ai-engine/config.py) 读取本机环境变量：

| 能力 | 配置项 | 默认值/说明 |
|---|---|---|
| 发单/回单业务 Agent | `DEEPSEEK_API_KEY` | DeepSeek Key，兼容兜底 `LLM_API_KEY` |
| 发单/回单业务 Agent | `LLM_BASE_URL` | `https://api.deepseek.com/v1` |
| 发单/回单业务 Agent | `LLM_MODEL` | `deepseek-chat` |
| SunPilot/PageAgent | `ALI_API_KEY` | 阿里云百炼 DashScope Key |
| SunPilot/PageAgent | `PAGE_AGENT_LLM_BASE_URL` | `https://dashscope.aliyuncs.com/compatible-mode/v1` |
| SunPilot/PageAgent | `PAGE_AGENT_LLM_MODEL` | `qwen3.7-plus` |
| MySQL | `MYSQL_ROOT_PASSWORD` | 用于自动拼接本地 `ticket_agent` 连接串 |
| MySQL | `DATABASE_URL` | 可选；连接非本地库时才手动配置 |

Windows 下，`config.py` 会先读当前进程环境变量；如果没有，再读 User/Machine 环境变量。因此推荐把 `DEEPSEEK_API_KEY`、`ALI_API_KEY`、`MYSQL_ROOT_PASSWORD` 配成系统级环境变量，然后重启后端。

配置入口在代码中已经固定，不需要每次启动时在终端手动设置 Key。后端进程启动后再修改系统环境变量，需要重启后端才能生效。

## 3. 启动后端

```powershell
cd C:\Users\heyunhui\PyProjects\TicketAgent
uv run python -m uvicorn main:app --app-dir ai-engine --reload --port 8000
```

启动后检查：

- Swagger: `http://127.0.0.1:8000/docs`
- 工单 API: `http://127.0.0.1:8000/api/tickets`

默认数据库为 MySQL。若未显式设置 `DATABASE_URL`，项目会使用 `MYSQL_ROOT_PASSWORD` 自动拼接：

```text
mysql+asyncmy://root:***@127.0.0.1:3306/ticket_agent?charset=utf8mb4
```

## 5. 演示主流程

1. 打开 `http://127.0.0.1:5173/tickets`。
2. 在“通话发单工作区”选择通话样本。
3. 点击“生成发单草稿”，由发单 Agent 产出标准工单草稿。
4. 在右侧 SunPilot 输入任务，或点击“执行建议”，让 PageAgent 可见填单并提交。
5. 进入工单详情后，启动 AI 处理。
6. 后端五类业务 Agent 完成分类、抽取、工具调用、升级判断和回单建议。
7. SunPilot 接收后端 observation，填入回单草稿、定位证据和审计区域。
8. 坐席人工复核后再结案。

边界：自动流程不能直接结案；结案必须走主系统人工复核动作。

## 6. SunPilot 模型与 Key

SunPilot/PageAgent 的真实 Key 仍由后端读取 `ALI_API_KEY`。前端只访问本地后端代理：

```text
POST http://127.0.0.1:8000/api/llm/proxy/chat/completions
```

右侧 SunPilot 输入框底部的 `Key` 按钮用于临时更新当前后端进程内的 PageAgent Key/模型；适合演示现场临时切换。长期配置仍建议放系统级环境变量。

## 9. 关闭服务

在前后端终端中按 `Ctrl+C`。

端口被占用时：

```powershell
netstat -ano | findstr :8000
netstat -ano | findstr :5173
```

确认 PID 属于本项目临时服务后再结束：

```powershell
taskkill /PID <PID> /F
```

## 10. 常见问题

**DeepSeek 或百炼 Key 无效？**

确认系统级环境变量存在，并重启后端。`DEEPSEEK_API_KEY` 用于业务 Agent；`ALI_API_KEY` 用于 SunPilot/PageAgent。

**MySQL 配置错误？**

确认 `MYSQL_ROOT_PASSWORD` 存在。优先让项目自动拼接本地连接串；只有连接非本地库时才设置 `DATABASE_URL`。

**前端空白或接口失败？**

确认后端在 `127.0.0.1:8000`，前端在 `127.0.0.1:5173`。

**需要调试接口？**

访问 `http://127.0.0.1:8000/docs`。
