> 模块 M PageAgent 演示版
> 更新日期：2026-07-22

## 1. 准备一次即可

项目根目录：

```powershell
cd C:\Users\heyunhui\PyProjects\TicketAgent

```

前端依赖：


依赖要求：Python 3.10+ 且低于 3.13、uv、Node.js 18+、npm 9+、MySQL/TDSQL。

## 2. 环境变量

本项目默认使用 MySQL，不再支持 SQLite。推荐把下面两个变量配置为系统级环境变量：

| 变量 | 用途 |
|---|---|
| `MYSQL_ROOT_PASSWORD` | 本地 MySQL root 密码，用于自动拼接 `ticket_agent` 连接串 |
| `ALI_API_KEY` | PageAgent 专用 Qwen/DashScope API Key |

后端启动时会优先读取当前 PowerShell 进程环境变量。若变量在系统级，建议在启动前显式同步到当前进程：

```powershell
$env:MYSQL_ROOT_PASSWORD=[Environment]::GetEnvironmentVariable("MYSQL_ROOT_PASSWORD","Machine")
$env:ALI_API_KEY=[Environment]::GetEnvironmentVariable("ALI_API_KEY","Machine")
```

不要手写 `DATABASE_URL`，除非你明确要连接非本地库。项目会用 `MYSQL_ROOT_PASSWORD` 自动拼接并 URL 编码：

```text
mysql+asyncmy://root:***@127.0.0.1:3306/ticket_agent?charset=utf8mb4
```

## 3. 启动后端

新开 PowerShell，在项目根目录执行：

```powershell
cd C:\Users\heyunhui\PyProjects\TicketAgent

$env:DB_BACKEND="mysql"
$env:MYSQL_ROOT_PASSWORD=[Environment]::GetEnvironmentVariable("MYSQL_ROOT_PASSWORD","Machine")
$env:ALI_API_KEY=[Environment]::GetEnvironmentVariable("ALI_API_KEY","Machine")

uv run python -m uvicorn main:app --app-dir ai-engine --reload --port 8000
```

启动后检查：

- Swagger: `http://127.0.0.1:8000/docs`
- 工单 API: `http://127.0.0.1:8000/api/tickets`

## 4. 启动前端

另开 PowerShell：

```powershell

```

打开：

- 企业工单系统: `http://127.0.0.1:5173/tickets`
- 旧版 fallback: `http://127.0.0.1:5173/legacy/tickets`

## 5. PageAgent 演示流程

1. 打开 `http://127.0.0.1:5173/tickets`。
2. 在“通话发单工作区”选择 `call-001` 或默认样本。
3. 点击“生成发单草稿”，确认草稿出现客户号、客户姓名、卡尾号、业务场景和发单内容。
4. 右侧 PageAgent Panel 会收到 bridge observation，并自动可见填表、提交标准工单。
5. 进入工单详情后，点击或让 PageAgent 点击“启动 AI 处理”。
6. Panel 会展示 Classifier、Intake、Escalation、Resolution、Notification 等后端 Agent 进度。
7. 后端完成后，PageAgent 会填入客户回单、定位证据编号，并停在人工复核/结案节点。

边界：自动流程不直接结案；结案仍必须人工复核后走主系统结案动作。


## 8. 关闭服务

在前后端终端中按 `Ctrl+C`。

端口被占用时：

```powershell
netstat -ano | findstr :8000
netstat -ano | findstr :5173
```

确认 PID 属于本项目临时服务后再结束：

```powershell
taskkill /PID <PID> /F
```

## 9. 常见问题

**PageAgent 报 `invalid_api_key`？**

后端进程没有拿到有效 `ALI_API_KEY`。如果 key 在系统级环境变量，重启后端前执行：

```powershell
$env:ALI_API_KEY=[Environment]::GetEnvironmentVariable("ALI_API_KEY","Machine")
```

**MySQL 配置错误？**

确认 `MYSQL_ROOT_PASSWORD` 存在，并优先让项目自动拼接连接串，不要手写 `DATABASE_URL`。

**前端空白或接口失败？**

确认后端在 `127.0.0.1:8000`，前端在 `127.0.0.1:5173`。

**需要更多 API 调试？**

访问 `http://127.0.0.1:8000/docs`，Swagger 里有所有工单、通话发单、AI 处理、工具审计接口。
