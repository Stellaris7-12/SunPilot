## 二、快速启动

> 推荐方式：**只使用项目根目录 `.venv`**，并由 `uv` 统一管理 Python 依赖。  

### 1. 进入项目目录（PowerShell）

```powershell
cd C:\Users\heyunhui\PyProjects\TicketAgent
```

### 2. 初始化 Python 环境

首次执行：

```powershell
uv sync
```

后续如果只是启动项目，不需要手动激活 `.venv`。所有后端命令都用 `uv run` 执行。

如果需要检查后端依赖是否可用：

```powershell
uv run python -c "import fastapi, openai, aiosqlite; print('backend deps ok')"
```

### 3. 配置 LLM API Key

PowerShell 当前终端临时配置：

```powershell
$env:DEEPSEEK_API_KEY="sk-your-deepseek-api-key"
```

可选配置（有默认值，按需设置）：

```powershell
$env:LLM_BASE_URL="https://api.deepseek.com/v1"
$env:LLM_MODEL="deepseek-chat"
$env:LLM_TIMEOUT="30"
$env:LLM_MAX_TOKENS="2000"
$env:LLM_TEMPERATURE="0.1"
$env:PORT="8000"
```

### 4. 启动后端

从项目根目录执行：

```powershell
uv run python -m uvicorn main:app --app-dir ai-engine --reload --port 8000
```

启动成功后输出：
```
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Starting Credit Card Multi-Agent System...
INFO:     Database initialized. 5 agents registered.
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8000
```

### 5. 安装前端依赖并启动

新开一个终端：

```powershell
cd C:\Users\heyunhui\PyProjects\TicketAgent\frontend
# 初次需安装npm
npm.cmd install
# 后续可直接运行
npm.cmd run dev
```

> PowerShell 里直接执行 `npm run dev` 可能触发 `npm.ps1` 执行策略报错，所以推荐使用 `npm.cmd`。

浏览器打开 **http://localhost:5173**。

### 6. Bash / Git Bash 等价命令

如果使用 Bash / Git Bash：

```bash
cd /c/Users/heyunhui/PyProjects/TicketAgent
```

## 三、关闭服务

### 方法一：Ctrl+C（正常关闭）

在运行前后端的终端窗口中，按 **`Ctrl + C`** 即可停止该服务。

> **Windows CMD 注意**：如果 Ctrl+C 无响应（通常因为存在活跃的 SSE 长连接），按 **`Ctrl + Break`**（Pause 键），或使用方法二强制终止。

### 方法二：强制终止端口（Ctrl+C 无效时）

```bash
# 查看占用 8000 端口的进程 PID
netstat -ano | findstr :8000

# 记下输出最后一列的 PID，强制终止（替换为实际 PID）
taskkill /PID 12345 /F

# 或一步到位
for /f "tokens=5" %a in ('netstat -ano ^| findstr :8000 ^| findstr LISTENING') do taskkill /PID %a /F
```

### 恢复初始状态

如果想重置所有工单到初始状态，需要删除本地 SQLite 运行库文件并重启后端。该操作会清空本地处理记录，执行前请先确认。

PowerShell 命令如下：

```powershell
Remove-Item .\ai-engine\data\tickets.db -ErrorAction SilentlyContinue
Remove-Item .\ai-engine\data\tickets.db-shm -ErrorAction SilentlyContinue
Remove-Item .\ai-engine\data\tickets.db-wal -ErrorAction SilentlyContinue
```

### 关于虚拟环境

使用推荐的 `uv run ...` 启动方式时，不需要手动激活或停用虚拟环境。`uv` 会自动使用项目根目录 `.venv`。

### Demo 场景 1：优惠券补发（低风险，全自动）

1. 左侧点击工单 **"活动达标未收到餐饮优惠券"**

3. 点击 **"AI智能处理"** 按钮
4. 实时观察 Agent Trace 时间线：
   ```
   🟢 意图识别Agent → 已识别为补发优惠券场景（置信度 93%）
   🟢 字段抽取Agent → 已提取 4/4 个有效字段
   ```

5. AI 处理完成 → 回单草稿出现在编辑器中
6. 人工审核草稿内容，可修改
7. 点击 **"人工复核并结单"** → 工单状态变为"已结单"

### Demo 场景 2：地址修改（中风险，需人工确认）

6. 工具调用完成 → 回单草稿生成
7. 审核 → 结单

> 如果点击"拒绝（转人工）"，工单将升级为人工处理，不执行工具调用。

### Demo 场景 3：交易争议（高风险，自动升级）

1. 点击工单 **"客户反馈199元交易非本人消费"**

## 七、架构设计要点

### 5个Agent协作流水线

```
工单输入 → IntentAgent（意图识别）
         → ExtractAgent（字段抽取）
         → VerifyAgent（审核校验）
         → ToolCallingAgent（工具调用）
         → ReplyAgent（回单生成）
         → 人工终审 → 结单
```

### 三级风险路由

| 风险等级 | 处理方式 | 示例场景 |
|----------|---------|---------|
| **LOW** | 全自动5Agent管线 → 人工终审 | 优惠券补发 |
| **MEDIUM** | 校验后暂停 → 人工确认 → 继续执行 → 人工终审 | 地址修改 |
| **HIGH** | 跳过工具+回复 → 直接升级人工处理 | 交易争议 |

### A2A-lite Agent Card

每个Agent通过 `agent_cards.json` 自描述能力、输入输出Schema、依赖关系和运行约束。编排器在启动时加载所有卡片，按拓扑排序决定执行顺序。

### SSE流式Trace

前端通过 `EventSource` 连接 `/api/tickets/{id}/ai-process-stream`，实时接收每个Agent的执行状态：
- `agent_start` → Agent开始工作
- `agent_thinking` → Agent进度消息  
- `agent_complete` → Agent完成（含结果+耗时）
- `workflow_paused` → 管线暂停等待人工确认
- `workflow_complete` → 管线完成（含完整结果）

## 八、常见问题

**Q: 启动后端报 `AuthenticationError: 401`？**

A: 未配置有效的 LLM API Key。请按第二章设置 `DEEPSEEK_API_KEY` 环境变量。支持 DeepSeek 和 Qwen（OpenAI 兼容接口）。

**Q: 前端页面空白？**

A: 确认后端已启动在 `localhost:8000`。前端通过 Vite proxy 将 `/api` 请求转发到后端。

**Q: 如何重置工单状态？**

A: 删除 `ai-engine/data/tickets.db`、`ai-engine/data/tickets.db-shm`、`ai-engine/data/tickets.db-wal` 后重启后端，系统会重新创建数据库并加载种子数据。删除前请确认你不需要保留本地处理记录。

**Q: 如何切换 LLM 模型？**

A: 设置 `LLM_MODEL` 环境变量，例如 PowerShell 中执行 `$env:LLM_MODEL="qwen-plus"`。

**Q: 如何添加新的工单场景？**

A: 1) 在 `data/tickets.json` 中添加工单数据；2) 在 `agents/intent_agent.py` 的 Prompt 中添加新的意图类型；3) 在 `agents/extract_agent.py` 中添加对应的字段模板；4) 如需新工具，在 `data/tools.json` 中添加工具定义。
