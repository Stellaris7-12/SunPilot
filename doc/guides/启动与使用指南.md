# TicketAgent 启动与使用指南

> 版本：I2/I3 工单流转与工具补全版  
> 更新日期：2026-07-21

## 一、环境要求

| 依赖 | 要求 | 说明 |
|---|---|---|
| Python | 3.10+，低于 3.13 | 后端 FastAPI 与 Agent 链路 |
| uv | 建议使用当前项目锁文件 | Python 依赖管理 |
| Node.js | 18+ | 前端 Vue/Vite |
| npm | 9+ | 前端依赖与构建 |
| MySQL/TDSQL | MySQL 兼容协议 | 主演示数据库 |

当前数据库口径：

- 主演示环境推荐使用 MySQL/TDSQL。
- SQLite 只作为本地测试兼容或 fallback。
- 前端不直接连接数据库；前端访问 `http://localhost:8000/api`，后端再通过 Repository 访问 MySQL/TDSQL 或 SQLite。
- I2/I3 新增的工单写操作、状态流转、操作日志、Mock 业务域数据和工具审计均通过后端 Repository 访问数据库。

## 三、数据库配置

### MySQL/TDSQL 模式（推荐演示）

不要把真实密码写入仓库。当前约定 MySQL 用户为 `root`，密码来自系统环境变量 `MYSQL_ROOT_PASSWORD`。

PowerShell 临时配置：

```powershell
$env:MYSQL_ROOT_PASSWORD="your-password"
$env:DB_BACKEND="mysql"
$env:DATABASE_URL="mysql+asyncmy://root:${env:MYSQL_ROOT_PASSWORD}@127.0.0.1:3306/ticket_agent?charset=utf8mb4"
```

如果系统环境变量里已经有 `MYSQL_ROOT_PASSWORD`，只需要设置：

```powershell
$env:DB_BACKEND="mysql"
$env:DATABASE_URL="mysql+asyncmy://root:${env:MYSQL_ROOT_PASSWORD}@127.0.0.1:3306/ticket_agent?charset=utf8mb4"
```

MySQL/TDSQL schema 文件：

```text
ai-engine/migrations/mysql/001_i1_schema.sql
```

该 DDL 会创建/更新 `ticket_agent` 演示库结构，文件名沿用 I1，但当前运行链路已覆盖 I2/I3 需要的工单流转、操作日志和 Mock 业务域数据。执行前确认目标库是本地演示库或可重建沙盘库，不要指向真实业务库。

### SQLite fallback

如只想本地轻量测试：

```powershell
$env:DB_BACKEND="sqlite"
$env:DATABASE_PATH="C:\Users\heyunhui\PyProjects\TicketAgent\ai-engine\data\tickets.db"
```

不设置 `DB_BACKEND` 时，代码默认使用 SQLite。

## 九、使用流程

1. 打开 `http://localhost:5173/tickets`。
2. 在列表页可按信用卡二线业务队列、状态、优先级、处理人和逾期条件筛选工单，也可以新建演示工单。
3. 点击工单进入详情页，查看客户号、脱敏手机号、卡尾号、分类、优先级、处理人和 SLA。
4. 详情页主系统按钮支持编辑标题、指派、保存回单草稿、作废、重开和结案；所有写操作都会记录到 `/operations`。
5. 点击右侧 Agent Copilot 的 AI 辅助动作，生成处理建议或定位证据/缺失字段。I3 会在 `pending_info` 前先尝试通过只读 Mock 工具补全客户、卡片、交易、权益或申请字段。
6. 中风险动作必须通过 `/confirm-action` 人工确认。
7. 结案必须由主系统按钮触发 `/close`，并回写最终回单和结案时间。

注意：Agent Copilot 只做辅助，不直接保存、结案、转派或覆盖主系统状态。

## 十一、常见问题

**Q: 前端页面为空或请求失败？**

A: 确认后端运行在 `localhost:8000`。前端 API 客户端默认访问 `http://localhost:8000/api`。

**Q: 为什么我启动后还是 SQLite 数据？**

A: 不设置 `DB_BACKEND` 时默认就是 SQLite。启动后端前设置 `DB_BACKEND=mysql` 和 `DATABASE_URL`。

**Q: MySQL 中没有 30 条干净演示工单？**

A: 确认连接的是 `ticket_agent` 演示库，然后执行：

```powershell
$env:DB_BACKEND="mysql"
$env:DATABASE_URL="mysql+asyncmy://root:${env:MYSQL_ROOT_PASSWORD}@127.0.0.1:3306/ticket_agent?charset=utf8mb4"
.venv\Scripts\python.exe ai-engine\scripts\reset_demo_data.py --confirm-reset-demo
```

**Q: I2/I3 是否已经在真实 MySQL/TDSQL 上全量 smoke？**

A: 当前代码路径支持 MySQL/TDSQL，I1 MySQL smoke 已有 `RUN_MYSQL_SMOKE=1` 验证命令；I2/I3 本轮主要通过 SQLite 临时库 smoke 验证。若要做主演示库验收，先确认 `DATABASE_URL` 指向可重置沙盘库，再运行重置脚本和 I2/I3 smoke。

**Q: 能否直接用工单编号打开详情？**

A: 可以。`/tickets/:id` 同时支持内部 id 和可见工单编号，例如 `/tickets/T202607210001`。

**Q: `closedLoopSuccessRate` 是真实生产结案率吗？**

A: 不是。它当前表示“状态/预期结果匹配率”，用于工程回归评测，不代表真实客户结案率。

**Q: LLM 返回 401 或鉴权失败？**

A: 设置有效的 `DEEPSEEK_API_KEY`，或确认 `LLM_BASE_URL`、`LLM_MODEL` 与当前供应商兼容。
