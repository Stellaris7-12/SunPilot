# 数据库配置与 I1 验证

## 目标

模块 K 后的数据库路线是：主演示环境和 smoke 测试均使用 MySQL 或 TDSQL，不再保留 SQLite 运行路径。当前代码已经收口 Repository 边界，并提供 MySQL-only smoke、MySQL/TDSQL DDL 和受控演示数据重置脚本。

## 配置

真实密码不要写入仓库。开发/演示库为 `ticket_agent`，smoke 测试库为 `ticket_agent_test`。Windows PowerShell 示例：

```powershell
$env:MYSQL_ROOT_PASSWORD="your-password"
$env:DB_BACKEND="mysql"
$env:DATABASE_URL="mysql+asyncmy://root:${env:MYSQL_ROOT_PASSWORD}@127.0.0.1:3306/ticket_agent?charset=utf8mb4"
```

## MySQL/TDSQL DDL

DDL 文件：

```text
ai-engine/migrations/mysql/001_i1_schema.sql
```

执行 DDL 会修改数据库 schema，运行前需要确认目标库是本地演示库或可重建沙盘库，不要指向真实业务库。

## 演示数据

业务演示种子：

```text
ai-engine/data/tickets.json
```

评测样本：

```text
ai-engine/data/evaluation_samples.json
```

二者必须隔离。演示种子用于页面、CRUD、状态流转和沙盘演示；评测样本只用于模块 E/F 回归。

受控重置演示库：

```powershell
.venv\Scripts\python.exe ai-engine\scripts\reset_demo_data.py --confirm-reset-demo
```

该脚本会清空当前 `DB_BACKEND` 指向的演示库中的工单、AI 结果、Trace、工具审计和操作日志，再从 `tickets.json` 导入干净业务工单；不会修改评测样本。运行前必须确认目标库是本地演示库或可重建沙盘库。

## 当前边界

- MySQL/TDSQL 运行态 adapter 已通过 `SQLAlchemy + asyncmy` 接入；Alembic 依赖已预留，当前首版迁移仍使用显式 DDL。
- Repository 已收口业务访问层，避免新增业务逻辑散落数据库细节。
- `.env.example` 只提供占位连接串；真实密码来自 `MYSQL_ROOT_PASSWORD`。
