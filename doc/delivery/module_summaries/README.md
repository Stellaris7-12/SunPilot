# 模块总结目录

本目录用于存放各模块的交付总结，重点说明“模块干了什么、原理是什么、核心内容是什么、结果表现是什么”，不写开发流水账，也不重复 `doc/planning/task_plan.md` 中已经完成的验收清单。

## 文件命名

建议使用英文路径、中文正文：

```text
module-a-backend-contract.md
module-b-agent-orchestration.md
module-c-resolution-tool-audit.md
module-d-notification-closure.md
module-e-evaluation-dataset.md
module-f-agent-evaluation.md
module-g-frontend-workbench.md
```
